# Cupboard

Cupboard 是一款專為 Minecraft PvP 伺服器設計、類似 Rust 領地保護系統的插件。玩家可以透過放置一塊金磚（Cupboard）來保護自己的建築與物品，同時仍保留像 TNT 破壞這類具攻擊性的機制——一切都可以依照你伺服器的平衡與風格自由設定。

**支援平台：** Spigot、Bukkit、Paper、Purpur、Leaves、Folia、Luminol

---

## 簡介

Cupboard 加入了類似 Rust 的領地保護系統。玩家只要放置一塊金磚，就能以該金磚為中心，宣告一個 19×19×19 的立方體保護區域。在這個範圍內，未經授權的玩家無法放置或破壞方塊。而像 TNT 這類爆炸物依然有效（可設定），讓攻城掠地的玩法得以保留，同時不破壞 Minecraft 原有的核心機制。

---

## 特色功能

- 以金磚為中心的區域保護（19×19×19）
- 透過右鍵點擊金磚進行權限授權
- 可設定是否阻擋 TNT 在保護區內造成傷害
- 預設阻擋苦力怕（Creeper）在保護區內造成的爆炸傷害
- 防止地獄傳送門：
  - 被固體方塊擋住
  - 被岩漿覆蓋
  - 被非玩家實體使用
- 可自訂地獄傳送門的搜尋半徑
- 殭屍豬人會掉落地獄疣
- 可設定世界邊界
  - 邊界可隨時間縮小
  - 可調整地獄與終界的邊界縮放比例
- TNT 可破壞黑曜石、水與岩漿（可設定）
- 完整支援語言檔翻譯（英文、繁體中文等）

---

## 保護內容說明

**受到保護的行為：**
1. 未授權玩家無法放置或破壞方塊
2. 未授權玩家無法使用石質壓力板或石質按鈕
3. 生物無法觸發石質壓力板
4. 苦力怕無法在保護區內破壞方塊

**不受保護的行為：**
1. 未授權玩家可以放置 TNT，且會立即點燃（可設定）
2. 未授權玩家仍可使用：
   - 木質壓力板與按鈕
   - 箱子
   - 盔甲座
3. TNT 可破壞保護區內的方塊（可設定）

---

## 指令與權限

- `/kill` — 讓玩家可以自殺（若不慎被困在保護區內時很有用）
- 不需要額外的權限系統
- 創造模式下的管理員（OP）會自動略過保護限制

---

## 安裝方式

1. 下載插件的 `.jar` 檔案
2. 放入伺服器的 `plugins/` 資料夾
3. 重新啟動伺服器
4. 編輯 `config.yml` 與語言檔（`locales/` 資料夾）來調整設定

---

## 語言檔

語言檔位於 `plugins/Cupboard/locales/` 資料夾內。可在 `config.yml` 的 `locale` 欄位中設定要使用的檔名（不含 `.yml`）。內建語言檔如下：

| 語系代碼 | 語言 |
|---|---|
| `en-EN` | 英文（預設） |
| `zh` | 中文（繁體） |
| `zh_TW` | 中文（台灣，繁體） |
| `zh_HK` | 中文（香港，繁體） |

若要新增其他語言翻譯，只需在 `locales/` 資料夾中建立一個新的 `.yml` 檔案，使用相同的鍵值（key），再將 `config.yml` 中的 `locale` 設定為該檔案名稱即可。

## 相容性

本插件已測試並相容於以下平台：

- Spigot
- Bukkit
- Paper
- Purpur
- Leaves
- Folia
- Luminol

**版本備註：**
- 以 Minecraft 1.20 API 為基準（1.20.x 及之後版本）
- 需要 Java 17 以上版本
- Folia 透過相容排程層（compatibility scheduler layer）提供支援
- Purpur 為 Paper 的分支版本，共用相同的 Bukkit/Paper API 介面
